如果你平时就在玩 `Codex`、`Claude Code` 这类 Agent，我建议你一定去看一眼 `UZI-Skill`。

因为这玩意不是“AI 帮你看看股票”这么简单。

它上来就是一整套狠活：

- `22` 维分析
- `17` 种机构方法
- `51` 位不同风格的“投资大佬评审团”
- 最后还给你一份真的像回事的结果页和分享卡

我第一眼看到的时候，脑子里其实就一句话：

**这不就是把“看票”这件事，硬生生做成了一个能跑的 skill 吗？**

而且它最容易让人上头的地方，不是标题，而是你点进去之后发现它真有成品。

不是那种“概念讲得天花乱坠，结果只会吐几段空话”的项目。

而是那种你看完截图、看完命令、看完报告之后，会很容易冒出一个念头：

**我也想装起来跑一遍。**

## 最先把我炸到的，就是它最后给你的效果

因为它不是那种你问一句，它回你一段话，就完了。

它最后给你的东西，是一整套结果页。

先看这个综合页：

![UZI-Skill 综合评分示意](/img/posts/uzi-hero-score.png)

你看这种画面就知道，它明显不是在做“聊天陪伴型 AI”。

它做的是另外一件事：

**把一只股票分析完之后，给你整理成一份像模像样的结果。**

里面有：

- 标的
- 分数
- 倾向判断
- 风险提示
- 后续还能往下翻的分析内容

这个体验其实挺关键的。

因为很多 AI 工具的问题不是“不聪明”，而是输出太散。

你问它一堆问题，最后得到一坨聊天记录，根本不想回看，更别说发给别人。

`UZI-Skill` 至少在这件事上，是认真做了“成品感”的。

## 第二个让我上头的点，是它那个评审团机制

这项目最容易传播起来的地方，肯定就是这个。

它把“多视角分析”这件事，做成了一个很直观的东西。

比如这张：

![UZI-Skill 评审席示意](/img/posts/uzi-jury-seats.png)

说白了，这个设计就很适合种草。

因为普通人一看到就懂了：

- 哦，这不是单一结论
- 哦，这是很多不同风格的人在一起看
- 哦，原来它是把“观点分歧”也做出来了

这玩意的传播力，比“我们用了 22 个维度分析、17 种机构方法”这种话术强太多了。

不是因为后者不重要，而是因为前者更容易让人一下子明白：

**这个工具做出来的东西，确实跟普通 AI 回答不太一样。**

## 它不是只有花活，往下翻也真有内容

如果它只有一个总分，或者只有一个“建议关注”，那我大概率不会专门写一篇来安利。

但它不是。

它后面还会继续展开很多深度卡片。

![UZI-Skill 深度卡片示意](/img/posts/uzi-deep-scan.png)

这个我觉得很重要。

因为很多人对这类工具天然有一个担心：

“你不会最后就给我一个结论，然后让我自己脑补过程吧？”

`UZI-Skill` 至少从公开产物看，不是这个路子。

它更像是在说：

- 我不只给你结论
- 我还给你拆开的分析层
- 你可以继续顺着往下看

这就让它更像一个“研究工作台”，而不是一个“只会下判断的聊天框”。

## 那它到底能拿来干嘛

如果你把它当成一个平时能拿来用的 skill，我觉得主要有 4 个用法。

### 1. 拿它跑完整个股分析

这肯定是主玩法。

README 里最核心的命令就是：

```text
/stock-deep-analyzer:analyze-stock 贵州茅台
```

这条命令的意思很简单：

你给它一只票，它帮你把一整套分析流程先跑一遍。

如果你平时自己看票的习惯是：

- 看点基础面
- 看点估值
- 看点情绪
- 再看看别人怎么理解

那这个东西本质上就是把这些动作尽量揉到一个入口里。

### 2. 拿它跑专项命令

这个也很实用。

因为很多时候你不是想来一份总报告，你只是想单看某一块。

比如：

```text
/stock-deep-analyzer:dcf 600519
/stock-deep-analyzer:quick-scan 002217
/stock-deep-analyzer:scan-trap 002217
/stock-deep-analyzer:earnings 002273
/stock-deep-analyzer:catalysts 002273
```

这几个命令看一眼就知道分别是干嘛的：

- `dcf`：看估值
- `quick-scan`：先快扫一遍
- `scan-trap`：先排雷
- `earnings`：看财报
- `catalysts`：看催化剂

我个人其实挺喜欢这种设计。

因为它说明这个项目不是只有一个“大而全入口”，而是真的考虑到了“平时我会怎么用它”。

### 3. 拿它做可分享的结果

这个也是我觉得它比很多工具更适合种草的地方。

它不是只给你一个长报告，它还有这种分享卡：

![UZI-Skill 分享卡片示意](/img/posts/uzi-share-card.png)

你别小看这种图。

很多时候一个工具到底能不能被持续用下去，取决于它最后给你的东西，能不能：

- 自己回看
- 发给别人
- 留着做后续参考

这类“成品感”做出来之后，整个体验就会从“我试过一个命令”变成“我手里真的拿到一个成果”。

### 4. 拿它塞进 Agent 工作流里

如果你平时就在用 `Codex`、`Claude Code`、`Cursor` 这类东西，那 `UZI-Skill` 的吸引力会更大。

因为它不是一个孤零零的网页产品，而是个 skill。

也就是说，你可以把它理解成：

**给你的 Agent 加一套股票研究能力。**

这个脑补一出来，味道就对了。

它不是让你换工作流，而是直接长在你原来的工作流上。

## 那到底怎么用

这部分我尽量写得最接地气。

### 如果你用 Claude Code

README 里的方式是：

```text
/plugin marketplace add wbh604/UZI-Skill
/plugin install stock-deep-analyzer@uzi-skill
```

装完之后，你最常用的大概率就是这几个：

```text
/stock-deep-analyzer:analyze-stock 贵州茅台
/stock-deep-analyzer:quick-scan 002217
/stock-deep-analyzer:scan-trap 002217
/stock-deep-analyzer:dcf 600519
```

### 如果你用 Codex

最简单的做法，其实就是直接让 Codex 按仓库说明去装：

```text
请按照 https://raw.githubusercontent.com/wbh604/UZI-Skill/main/.codex/INSTALL.md 的指引安装 UZI-Skill，然后帮我深度分析 贵州茅台。
```

这种 skill 最省脑子的用法，本来就不是你自己一行行研究怎么配，而是直接让 Agent 干。

### 如果你想直接用命令行

那也可以。

仓库给的公开方式是：

```bash
git clone https://github.com/wbh604/UZI-Skill.git
cd UZI-Skill
pip install -r requirements.txt
python run.py 贵州茅台
```

我自己本地也直接拉起过：

```bash
python run.py --help
python run.py 600519 --no-browser
```

至少从入口上看，它不是那种“README 写得很漂亮，实际命令一跑就死”的项目。

## 如果你问我，最推荐怎么上手

我会建议你别一上来就冲最重的玩法。

最省时间的姿势其实是：

1. 先跑 `quick-scan`
2. 觉得有意思再跑 `analyze-stock`
3. 真要看某个点，再去用 `dcf`、`earnings`、`catalysts`

这样最顺。

因为你会先知道：

- 这玩意是不是你的菜
- 你喜不喜欢它的输出风格
- 你有没有必要继续花更多时间

## 这东西适合谁

我觉得它特别适合下面这几类人：

1. 平时就在玩 `Codex`、`Claude Code`、`Cursor` 的人
2. 对股票研究流程感兴趣，但又不想自己拼整套流程的人
3. 喜欢“把 Agent 真正用到场景里”的人

如果你完全不碰命令行、不碰插件、不碰仓库，那这玩意对你来说还是有门槛的。

如果你想找的是“直接告诉我明天买什么”，那我也不建议你按这个方向去理解它。

因为它更像： **研究助手**

而不是： **买卖指令机**

## 最后还是得把边界讲清楚

虽然我这篇是在种草，但边界还是得留。

项目作者自己在 README 里写得很明确：

> 本工具由 AI 模型基于公开数据生成分析报告。所有评分、建议、模拟评语均为算法输出，不代表任何真实投资者的实际观点。不构成投资建议，投资有风险，入市需谨慎。

所以我这里也保留同样的意思：

> 本文仅为工具效果、用途与上手方式分享，不构成任何投资建议，相关内容仅供参考。

如果你是把它当成一个“研究流程增强器”，那它挺值得试。

如果你是把它当成一个“替你做决定的机器”，那就别这么期待。
